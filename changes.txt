Session: 2025-10-19 (Part 5) - Prescription Tracking Without Auto-Add
----------------------------------------------------------------------

MAJOR CHANGE:

* Prescription Tracking No Longer Auto-Adds Inventory
  - Problem: Enabling prescription tracking was doubling inventory
    • Set "Current medication count" to 60
    • System added 60 more pills automatically
    • Ended up with 120 instead of 60
  - Solution: Prescription details now stored in MedicationReminder model
    • "Current medication count" field = exact starting inventory
    • Prescription metadata (pills per refill, total refills) stored but don't affect inventory
    • Only "Record Refill Pickup" button adds pills to inventory
    • No automatic prescription record creation

DATABASE CHANGES:

* Schema Version: 9 → 10
  - Added `prescriptionPillsPerRefill` column to medication_reminders table (default: 60)
  - Added `prescriptionTotalRefills` column to medication_reminders table (default: 5)
  - Migration MIGRATION_9_10 handles automatic upgrade
  - Prescription metadata now part of medication record, not separate refill record

TECHNICAL CHANGES:

* Modified Files:
  - MedicationReminder.kt: Added prescriptionPillsPerRefill and prescriptionTotalRefills fields
  - DatabaseMigrations.kt: Created MIGRATION_9_10
  - AppDatabase.kt: Updated version to 10, registered migration
  - MainActivity.kt: Forms now save/load prescription fields from reminder
  - MedicationReminderViewModel.kt: Simplified insert logic (removed auto-prescription creation)

* Behavior Changes:
  - insertWithPrescription no longer creates prescription records
  - Prescription fields stored directly in reminder
  - PrescriptionRefill records only created when explicitly recording pickups
  - Edit form loads prescription values from reminder.prescriptionPillsPerRefill/TotalRefills

USER WORKFLOW:

* Before:
  1. Enable prescription tracking
  2. Enter current count: 60
  3. System creates prescription + adds 60 pills
  4. Inventory becomes 120 (wrong!)

* After:
  1. Enable prescription tracking
  2. Enter current count: 60
  3. Enter prescription details (60 pills/refill, 5 refills)
  4. Inventory stays 60 (correct!)
  5. Later: Click "Record Refill Pickup" → inventory increases

Version 1.2 - Customizable Refill Period & Enhanced Calendar
------------------------------------------------------------

NEW FEATURES:

* Customizable Refill Period
  - Added ability to set custom refill period per medication (default: 30 days)
  - Users can now specify 30, 60, 90 day refill cycles or any custom period
  - New "Refill period (days)" input field in medication setup form
  - Refill period is configurable only when inventory tracking is enabled

* Multiple Future Refill Date Indicators
  - Calendar now displays up to 12 future estimated refill dates (orange dots)
  - Orange dots show estimated pharmacy pickup dates for the next ~1 year
  - Dots automatically shift when actual pickup occurs early or late
  - Period between dots remains constant based on your refill period setting

IMPROVEMENTS:

* Smart Refill Date Calculation
  - Orange dots now calculate from: lastActualPickupDate + (refillPeriodDays × N)
  - If you pick up 2 days early, all future dots shift 2 days earlier automatically
  - Refill estimates are no longer affected by taking/not taking medication
  - Provides accurate long-term planning for pharmacy visits

DATABASE CHANGES:

* Schema Version: 8 → 9
  - Added `refillPeriodDays` column to medication_reminders table
  - Migration MIGRATION_8_9 handles automatic upgrade
  - Default value: 30 days for all existing medications
